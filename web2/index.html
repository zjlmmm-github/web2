技术架构

系统由数据中心、云设计前端、云渲染端、管理后台四部分组成。
云设计端: 采用javasricpt、threejs开发, vue框架;
数据中心: 采用mysql数据库、C#写Webservice接口进行前后端通信, 主要部署在Win系统服务器上，也可部署在CentOS系统上；
云渲染端: 采用C#开发,自动与数据中心通信、自动更新数据、自动渲染;
后台管理: 采用javasricpt开发, vue框架;

系统环境

数据中心服务器: 操作系统: windows server 2016；数据库: mysql5.6.10
云渲染服务器: 操作系统windows server 2008 R2 Enterprise 64位

常见问题解答

常见的一些问题解答。帮助用户在使用开发平台过程中，更好的理解平台服务。

1. 免费版可以用来商业开发吗?

可以。在遵守国家法律法规前提下，免费版、商业版都可以用来二次开发,定制商业系统。同时也不得损害iHouse3D版权及利益.

2. 可以独立部署到自己服务器吗?

可以, 联系平台。我们将部署管理后台、数据中心、云渲染等到指定的服务器。

3. iHouse3D设计系统与Threejs的区别是什么?

ThreeJs是一款免费的网页3D开发引擎, iHouse3D是装修设计系统原型，方便用户快速开发自己的系统。iHouse3D前端也是基于Threejs开发。

4. 我没有服务器怎么办

iHouse3D平台提供管理后台,自己管理账号、管理模型资源. 提供云渲染服务, 提供良好的开发配套服务。

5. iHouse3D平台会使用企业在平台上的信息吗?

不会, iHouse3D只提供开发等纯技术服务， 我们鼓励用户独立部署到自己服务器上。

6. iHouse3D设计系统对模型有什么要求

如max、skectup、maya等常见建模工具都可以建模导出, 模型要求:简面模型、使用的贴图完整。
